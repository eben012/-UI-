# 🚨 다크패턴 UI 경험 - 교육 프로젝트

다크패턴(Dark Patterns)의 위험성을 직접 체험하고 학습하는 인터랙티브 웹사이트입니다.

## 📱 웹사이트 접속

🔗 **웹사이트 링크**: [https://eben012.github.io/-UI-/](https://eben012.github.io/-UI-/)

## 🎯 프로젝트 개요

이 프로젝트는 음식 배달 앱에서의 결제 과정을 통해 **6가지 다크패턴 유형**을 시연합니다.

### 🚨 3단계 시나리오

#### **[1단계: 장바구니 화면]**
- **특정옵션 사전선택** + **숨은 가격 결정**
- 멤버십 체크박스가 기본 ON 상태로 설정
- 작은 글씨로 멤버십 요금 표시

#### **[2단계: 결제 버튼 확인 팝업]**
- **잘못된 계층구조** + **속임수 질문**
- 기업이 원하는 선택(큰 빨간 버튼)과 사용자가 원하는 선택(흐린 회색 버튼) 구분
- "3,000원 할인을 포기하시겠습니까?" 메시지로 판단 흐리기

#### **[3단계: 최종 결제 승인]**
- **반복적 간섭** + **감정적 언어사용**
- 마지막 순간에 또 다른 팝업으로 구매 행위 방해
- "저는 혜택을 포기하고 돈을 더 내겠습니다" 같은 부정적 메시지

## 📚 다크패턴 유형 (6가지)

1. **특정옵션 사전선택** - 기본값으로 기업에 유리한 옵션 선택
2. **숨은 가격 결정** - 작은 글씨로 추가 요금 표시
3. **잘못된 계층구조** - 중요도를 시각적으로 왜곡
4. **속임수 질문** - 혼동을 유도하는 표현
5. **반복적 간섭** - 사용자의 의사 결정을 방해
6. **감정적 언어사용** - 죄책감이나 두려움 유도
